Day 3 — CriticalThe 2 biggest beginner mistakes in n8n — learn these now, save 10 hours▼
Read this before building anything elseThese 2 mistakes trip up 90% of beginners. Know them on day 3.
  • Mistake #1 — Set node overwrites everything: The "Set" node has two modes: "Keep Only Set" (default — DELETES all previous node data and only keeps what you define) and "Add" mode (keeps all previous data and adds new fields). Beginners use Set node to add one field, then wonder why all their Google Sheet data disappeared downstream. Rule: always switch Set node to "Add" mode unless you deliberately want to strip all other data. This single mistake wastes 2–3 hours when you first hit it.
  • Mistake #2 — Test webhook URL vs Production webhook URL: n8n gives you TWO different webhook URLs for every webhook trigger. The "Test" URL only works while you have the workflow open in the editor and have clicked "Test step". The "Production" URL only works when the workflow is ACTIVATED (the toggle in the top right is green). If you paste the Test URL into Stripe, Typeform, or any other service, it will stop working the moment you close the editor tab. Always use the Production URL for any real integration. Always activate the workflow before going live.
  • Where to find them: click on the Webhook trigger node → you'll see both URLs clearly labeled "Test URL" and "Production URL". Copy the Production URL. Paste it into the external service. Activate the workflow. Done.
These two mistakes are the #1 source of "why is my webhook not working??" posts on the n8n community forum. You now know the answer before you need it.

Day 5 — ExtraExecution history & debugging — how to diagnose any broken workflow▼
10am – 2pm · LEARNThe debugging skill that makes you look like an expert to clients
  • Every workflow run in n8n is logged in the Execution History. Go to any workflow → click "Executions" in the left panel. You'll see every run: success (green), failed (red), waiting (yellow). Click any failed run to see EXACTLY which node failed, what data it received, what error it threw.
  • How to debug systematically: (1) Click the failed execution. (2) Click the red node. (3) Read the error message — usually it says exactly what's wrong ("Invalid API key", "Field 'email' not found", "Rate limit exceeded"). (4) Fix it. (5) Use "Re-run from failed node" to re-test without triggering the whole workflow again.
  • Pin Data technique: click any node → click "Pin" → paste in fake test data. Now you can run just the downstream nodes without retriggering the trigger. Example: your workflow starts with a Stripe webhook, but you don't want to make a real payment every time you test. Pin fake Stripe data to the webhook node → run everything after it freely. This is the #1 time-saving technique for building faster.
  • When a client reports a bug: go to Executions → find the failed run around the time they reported the issue → click it → read the error → fix it → forward them the fix. This is professional client support. Takes 5 minutes when you know how.
  • Production monitoring habit: check your active client workflows' execution history once a week. Look for patterns — if the same node is failing repeatedly at the same time, there's a systemic issue (API down, data format changed, rate limit). Catch it before the client does.

Days 8–9 ExtraPagination — when APIs only give you partial data▼
10am – 2pm · LEARNSilent data loss. The API bug nobody tells beginners about.
  • Most APIs don't return all results at once. They return 100 records and say "there are 500 more — fetch the next page". If you don't handle pagination, your workflow silently processes only the first 100 records and you never know the rest exist.
  • Example: you build a workflow to sync 2,000 HubSpot contacts to Google Sheets. HubSpot returns 100 at a time. Without pagination, you get 100 contacts and think you got all 2,000. The client's sheet is 95% empty and they think your workflow is broken. It's not — you just didn't paginate.
  • n8n's built-in solution: the HTTP Request node has a "Pagination" tab. Click it, enable pagination, tell n8n where to find the "next page" cursor in the API response (usually something like response.next_cursor or response.offset). n8n will keep fetching until there are no more pages. This is a built-in feature — you don't need to write any code.
  • How to check if an API paginates: read its docs and look for words like "cursor", "offset", "next_page_token", "hasMore", "total_count". If any of these exist in the API response, you need pagination.
  • Build: use the HTTP Request node to pull ALL repositories from a GitHub user via the GitHub API (which paginates at 30 results). Enable n8n's pagination. Verify you get more than 30. This teaches you the skill in a safe environment.
Every time you connect to a new API, ask yourself: "does this API paginate?" If yes, enable pagination in the HTTP Request node before building anything else. It takes 30 seconds and prevents hours of confusion.

Days 10–11 ExtraWebhook Response node — how to talk back to the sender▼
10am – 2pm · LEARN + BUILDWithout this node, your chatbots and interactive webhooks feel broken.
  • When n8n receives a webhook, the sender (Stripe, WhatsApp, a chatbot platform) usually waits for a response back. If n8n doesn't respond within a few seconds, the sender gets a timeout error. This makes your integration look broken even when it worked.
  • The "Respond to Webhook" node solves this. Add it anywhere in your workflow to immediately send a response back to the caller — before your workflow finishes all its steps. Example: WhatsApp sends a message to your n8n webhook → n8n responds "Message received, processing..." in 200ms → then n8n does all the AI processing and database writes in the background.
  • Use case 1 — chatbot platforms (Typebot, Chatbase, custom bots): they send a user's message and wait for the AI reply. Your workflow must use "Respond to Webhook" with the AI's answer as the response body. Otherwise the chat widget times out.
  • Use case 2 — payment platforms: Stripe sends a payment webhook and waits for a 200 OK response within 5 seconds. If it doesn't get one, Stripe will retry the webhook up to 3 days. Add "Respond to Webhook" at the start: respond with 200 OK immediately → then do all your database/email work after. This prevents Stripe from sending duplicate events.
  • Build: create a webhook that receives a text message, immediately responds "Got it, generating your answer...", then calls the OpenAI API to generate a response. This simulates a real AI chatbot webhook. You'll use this pattern in every WhatsApp/Telegram AI bot you build for clients.

Week 2 ExtraCredentials & security — how to handle API keys safely for clients▼
10am – 2pm · LEARNThis is what separates professional freelancers from hobbyists
  • Read: n8n docs → "Credentials" section. Understand that n8n stores credentials encrypted — client API keys are never visible in plaintext in the workflow JSON.
  • NEVER hardcode API keys inside nodes using plain text. Always use n8n's Credentials system. Reason: if a client asks you to export the workflow as JSON and you hardcoded keys, those keys are visible in the exported file.
  • OAuth2 vs API Key vs Basic Auth: know the difference. OAuth2 (Google, HubSpot, LinkedIn) requires a 3-click authorization flow — you connect the account, n8n gets a token. API Key (OpenAI, Airtable) — you paste the key into the credential. Basic Auth (some legacy tools) — username + password.
  • For client work: always create a dedicated Google account / API key for each client. Never share your personal API keys. When a project ends, the client revokes their own keys and you still have yours.
  • Environment variables: for self-hosted n8n, store sensitive values (DB passwords, encryption keys) as environment variables in your server's .env file — not inside n8n workflows. Learn this before self-hosting in month 2.
Security mistake that kills freelance careers: sharing a workflow JSON with a client that has hardcoded API keys inside. Their developer opens the file, sees your OpenAI key, now they have free access to your account. Always use n8n Credentials. Always.

Week 3 ExtraLoop Over Items, Wait node, Split in Batches — handling bulk data▼
10am – 2pm · LEARN + BUILDMost client requests involve processing lists of things. These nodes handle that.
  • Loop Over Items node: processes each item in a list one by one. Use case: you have 50 leads in a Google Sheet → Loop Over Items → send a personalized email to each one. Without this node, n8n processes all items together (which usually creates one email, not 50).
  • Split In Batches node: processes items in groups. Use case: you have 500 rows to process but the API only allows 10 requests per second → Split In Batches (10 per batch) + Wait node (1 second delay) = no rate limit errors.
  • Wait node: pauses a workflow for a specific duration. Use case: drip email sequence — send email 1 immediately, wait 2 days, send email 2, wait 3 days, send email 3. This is how you build automated follow-up sequences.
  • Build: Google Sheet with 10 rows of fake leads → Loop Over Items → for each row, send a personalized email using the name and email from that row. This is one of the most common things clients actually need.
  • Build: 3-email drip sequence. Form submission → immediate welcome email → Wait 2 days → follow-up email → Wait 3 days → final CTA email. Real email marketing agencies charge $500–$800 for building this.
6pm – 10pm · BUILDWorkflow export, backup, and version control — protect your work

Days 53–60 — CriticalHow to receive international payments from Bangladesh — the practical guide▼
Set this up BEFORE you take your first client. A blocker if ignored.You need a way to receive USD/EUR from international clients into your BD bank account.
  • Option 1 — Payoneer (recommended for beginners): Free to sign up at payoneer.com. You get a US bank account number (routing + account number). Give this to international clients or Upwork. They pay in USD → it lands in Payoneer → you withdraw to your Dutch-Bangla Bank, BRAC, or any BD bank. Processing time: 2–3 business days. Withdrawal fee: ~$3 flat or 2% (whichever is higher). Payoneer also works as your Upwork payment method directly — connect them in Upwork settings. This is what most Bangladeshi freelancers use.
  • Option 2 — Wise (formerly TransferWise): Sign up at wise.com. You get a "borderless account" with USD, EUR, GBP account details. International clients pay into it. You convert to BDT and withdraw to your bank. Conversion rate is much better than Payoneer (closer to mid-market rate). Slight complexity: setting up for Bangladesh requires identity verification. Withdrawal to BD bank takes 1–3 days. Good for larger amounts where the better conversion rate saves meaningful money.
  • Platform-specific: Fiverr has its own Fiverr Revenue Card (Payoneer-powered). Connect Payoneer to Fiverr and withdrawals go straight to your Payoneer account. Upwork: in payment settings, add your Payoneer account as the withdrawal method. Direct clients: give them your Payoneer USD account number and they do a standard bank transfer.
  • Tax & regulatory note for Bangladesh: Freelance income from abroad is currently exempt from income tax in Bangladesh under the NBR freelancer incentive (verify this is still active — check the NBR website or ask a local accountant). You are required to bring the income through a proper banking channel (Payoneer/Wise → BD bank) and get a "remittance certificate" from your bank. Keep these. They're proof of legal foreign income if ever asked.
  • Do this now, before day 53: Sign up for Payoneer during Month 1, Week 1 — it sometimes takes 3–7 days to verify your account. Don't wait until you have a client ready to pay you and discover it takes a week to set up.

Days 53–60 — ExtraClient discovery call script — what to say to scope any project▼
10am – 2pm · LEARNA 20-minute discovery call done right turns a lead into a signed contract.

  • Opening (2 min): "Thanks for your time. I've read your project description — before I quote anything, I want to make sure I understand the problem exactly. Can you walk me through what your current process looks like, step by step?"

Week 13–14 ExtraQA & testing before client delivery — every time, no exceptions▼

Payoneer — payoneer.com (set up in Week 1)
How you receive international payments into your Bangladesh bank account. Free to sign up — you get a US bank account number. Upwork and Fiverr connect directly to Payoneer. Withdraw to any BD bank (BRAC, DBBL, etc.) in 2–3 days. Set this up in Week 1 — verification takes up to 7 days. Do not wait until you have a paying client.
