Day 12bWebflow Variables — the design system foundation you must know▼
10am – 2pm · LEARNCSS Variables in Webflow — critical for scalable, client-ready projects
  • Watch: "Webflow Variables" on Webflow University — released 2023, updated 2024. This is now a core feature and non-negotiable for any professional project.
  • What Variables do: Instead of manually setting the same blue (#1E6DFF) on 40 different elements, you set one Variable called "brand-blue" and apply it everywhere. When the client says "change our brand color" — you change 1 Variable and the entire site updates in seconds instead of hours.
  • Learn all 3 variable types: Color variables (brand palettes), Size variables (spacing scale, font sizes), and Variable Modes (light/dark theme switching without writing a line of code).
  • Variable Modes are especially powerful: one click switches a site from light mode to dark mode. Clients love this feature — it's a $300–$500 upsell on any project.
6pm – 10pm · BUILDRefactor your agency project to use Variables throughout
  • Create a Variables system in your agency project: 3 color variables (primary, secondary, neutral), 2 size variables (section-spacing, card-gap), and 1 font-family variable.
  • Replace all direct color values on your elements with Variables. It will take 1–2 hours and feel tedious. After this, change your primary brand color in 1 click and watch every element update. This is the "wow" moment that makes Variables click.
  • Add a second Variable Mode: a "dark mode" version of your color palette. Toggle between them on the canvas.
  • From now on: every project you build starts with a Variables setup. This is part of your professional standard.

Days 23–25SEO, GTM/Analytics, Custom Code, Forms, Image Optimization▼
10am – 2pm · LEARN5 professional skills that clients assume you know
  • Day 23 AM — SEO + Image Optimization + Google Tag Manager: Webflow SEO basics (meta tags, Open Graph, sitemap, canonical URLs). Then image optimization — the most overlooked skill: export images at 2x for retina but compress to under 200KB using Squoosh (squoosh.app, free browser tool) or TinyPNG. Use WebP format (Webflow now supports it natively). Without this your sites fail PageSpeed Insights — which clients check. Then Google Tag Manager (GTM): every business client needs GA4, Meta Pixel, Google Ads tracking. GTM is one code snippet added to Webflow (Page Settings → Custom Code → paste in <head>), then all tracking scripts are managed inside GTM without touching Webflow again. Watch: "Google Tag Manager tutorial for beginners" on YouTube (20 min). This is expected from any professional — not optional.
  • Day 24 AM — Custom Code embeds: Webflow's visual editor handles most things, but real client work constantly needs custom code. Learn the 4 places you add custom code in Webflow: (1) Site-wide <head> (Project Settings → Custom Code), (2) Site-wide before </body>, (3) Per-page <head>, (4) HTML Embed element for inline code. Common things that always need custom code: cookie consent banners (required by EU/UK law — use CookieYes or Cookiebot free scripts), live chat widgets (Crisp, Intercom — paste in <head>), heatmap tools (Hotjar, Microsoft Clarity — both free). Knowing how to add these makes you the complete solution, not just a site builder.
  • Day 25 AM — Webflow Forms + Finsweet Attributes: Native forms, Zapier integration (form → email + Google Sheet). Logic forms. Then Finsweet Attributes free toolkit: CMS Filter, CMS Sort, CMS Load More, Tabs CMS. These solve 80% of "can Webflow do this?" client questions without writing code.
6pm – 10pm · BUILDApply every skill the same evening
  • Day 23 PM: Run your portfolio piece #1 through PageSpeed Insights. Fix the top 2–3 issues (usually: compress images, add explicit width/height to images, defer non-critical scripts). Aim for 80+ score on mobile. Also: add GTM snippet to the project and set up a fake GA4 tag inside GTM to test the flow.
  • Day 24 PM: Add a CookieYes free cookie banner to your project via Custom Code. Then add the Hotjar tracking script (sign up free, get your script). Both should appear correctly when you preview the site. Practice finding where these go in Page Settings.

CRITICAL: Webflow Hosting Plans — must understand before talking to clientsHosting plan tiers — clients will ask and you must know the answer immediately
  • 5 tiers: Starter/Free (2-page published limit, free subdomain — practice only, no client work), Basic $18/mo (no CMS — static pages only), CMS $29/mo (blog, team pages, portfolio — most client sites need this), Business $49/mo (high traffic + 10,000 CMS items), E-commerce $42–$235/mo.
  • Most common beginner mistake: building a CMS-powered site then being confused why nothing publishes — the client is on the Basic plan which has no CMS. Always confirm the plan in discovery.
  • Your standard recommendation: CMS plan ($29/mo) for any site with a blog or dynamic content. Quote your build fee separately from hosting — the client pays Webflow monthly directly. State this clearly in every project agreement.
  • Free plan limit warning: The Starter plan only publishes 2 pages. You will hit this on Day 7 of this roadmap. For practice builds, stay on the free subdomain. For client projects, the client needs a paid plan — they buy it, you set it up.
6pm – 10pm · SET UP PAYMENTS NOWPayoneer + Wise — set these up today, before your first order arrives
  • Sign up for Payoneer (payoneer.com) — as a freelancer with Bangladesh address. Connect your Bangladeshi bank account (Dutch-Bangla, BRAC, Islami Bank all work). Approval takes 1–3 days. Both Fiverr and Upwork pay to Payoneer. Do this now, not when an order arrives — you need it ready.
  • Also sign up for Wise (wise.com) — lower withdrawal fees (~0.5%) compared to Payoneer (~1.5–2%). Set up both and connect both to Upwork's payment methods. Use Wise when possible, Payoneer as backup.
  • Fee reality check: On every $500 project, expect $7–$12 in combined withdrawal fees on top of the platform's cut. Factor this into how you price projects.
  • Get your TIN now: Free, takes 10 minutes at etaxnbr.gov.bd. You need it for Payoneer verification, your bank's foreign currency account, and your annual tax return filing. IT freelancing is tax-exempt until June 2027, but you still must have a TIN and file a zero-tax return each November 30.

CRITICAL: The complete client handoff and billing transfer processHow to end every project professionally — the step most beginners skip entirely
  • Step 1 — Transfer the project to the client's Webflow account: You built the site in your own Webflow workspace. The client needs it in theirs. The Editor is a simplified, non-destructive interface where clients can update text, images, and CMS content without touching your layouts, classes, or animations. Send them a Loom walkthrough video (5–8 minutes) showing how to use the Editor to update their own content. This single video prevents 80% of all "how do I change this?" support requests.

(2) A Google Drive folder with all images you used, any SVG icons, and logo files. (3) Login credentials for any third-party tools you connected (Zapier, Google Analytics, GTM).
  • Step 5 — Final delivery message: Send a short written summary: what was built, what hosting plan they need, how to use the Editor, and what to do if something breaks. Offer a 7-day free support window for any bugs or questions. After 7 days, any changes are billable. State this clearly.

Month 3 — ContractsClient contracts and Scope of Work — protect yourself before every project▼
Do this once — use it on every project foreverA simple SOW template prevents 90% of client disputes
  • Why you need a contract: Webflow projects are especially prone to scope creep — "can you add one more section?", "can we redesign the whole homepage?", "can you add a shop?" Without a written scope, clients believe revisions are unlimited and additions are free. A one-page SOW (Statement of Work) stops this from day one.
  • What your SOW must include: (1) Exact deliverables — list every page and feature by name. "5 pages: Home, About, Services, Blog (CMS), Contact." Not "a website." (2) Number of revision rounds included — "2 rounds of revisions. Additional rounds billed at $X/hr." (3) Timeline — project start date, delivery date, what happens if the client is late sending content. (4) Payment terms — 50% upfront, 50% on delivery. Non-negotiable. Always. Never start work without the deposit. (5) What's NOT included — hosting costs (client pays Webflow directly), copywriting, photography, SEO campaigns, ongoing updates after 7-day support window.
  • Simple contract tool: Use Bonsai (hellobonsai.com) — free tier lets you create and send contracts with e-signature. There are Webflow-specific contract templates in their library. Alternatively, copy a basic freelance contract template from Google Docs and customize it. It does not need to be a legal masterpiece — it just needs to exist so both sides have written expectations.
  • 50% upfront rule — follow this every time: Never do work without collecting 50% first. On a $500 project, that's $250 before you open Webflow. On Upwork this is handled by milestone payments — set milestone 1 at 50% before work starts. On direct clients, send a Payoneer payment link or Wise invoice. If a client refuses to pay upfront, that is a red flag. Move on.
  • Scope creep response script: When a client asks for something outside the SOW, your response is: "That sounds great — this would fall outside our current scope. I can add it as a new item for $X, or include it in a future retainer. Would either work?" Always friendly, never apologetic. You are not being difficult — you are running a business.
